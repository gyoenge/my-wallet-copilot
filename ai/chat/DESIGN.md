# Agentic AI 설계 

### Multi Agent System 

- 1. 사용자가 입력한 소비 내역 원본 파일을 점검하고, 해석에 용이한 형태로 변환하는 전처리 에이전트 
    - 소비 항목 카테고리 분류를 - 분류기 + 검증가(critic) 로 분리.
        검토 화면에 "불확실" 배지 + 신뢰도 순 정렬 + 추천 사유를 노출 -> 사용자가 의심스러운 것부터 빠르게 검수할 수 있도록.  
    - 이상 탐지 백그라운드 에이전트 
        업로드 시 탐지 에이전트가 구독 누수/중복/이례적 급증을 스캔 -> 대시보드에 동적 카드("점검 필요")로 푸시. 
    
- 2. 기본적인 월간 리포트를 정리하여 제공하는 분석 에이전트 
    - planner -> 섹션 작성가 -> 편집자 
        한 번에 "이번 달 리포트" 생성: 개요/카테고리/절약안 섹션을 병렬 작성 후 편집 통합. 

- 3. 사용자가 원하는대로 분석을 해주는 챗봇 
    - 단일 ReAct 챗을 코디네이터 (supervisor) 가 질문을 라우팅하는 구조로 
        - 패턴 분석가: 요일, 시간대, 카테고리 
        - 절약 코치: what-if, 절감 시뮬레이션 
        - 이상 탐지가: 구독 누수, 중복 결제, 급증
        - 예산 플래너: 예산, 예측 
      각 전문가의 시스템 프롬프트와 도구를 좁혀 답변의 품질을 높힌다. 다만 단순 질문엔 오버헤드가 있을 수 있어 라우팅이 핵심. 
    - 작업가와 세이비 페르소나를 분리 
        - 분석가 에이전트: 도구로 사실/숫자만 산출(중립). 
        - 세이비 에이전트: 그 결과를 "지갑 수호자" 톤으로 재작성. 
      정확성과 말투를 분리해 톤 일관성을 높이고, 페르소나 교체가 쉽도록 한다.
